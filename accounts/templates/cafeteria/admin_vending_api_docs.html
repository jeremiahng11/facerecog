Authentication

All requests must include a Bearer token in the Authorization header. The API key is configured via the VENDING_API_KEY environment variable on the server. 

Authorization: Bearer {{ api_key_preview }}

Deduct Credit

POST /cafeteria/api/vending/deduct/

Deduct credit from a staff member's cafeteria balance. The vending machine scans the staff member's QR code from their phone, then calls this endpoint with the QR token and purchase amount.

Request Headers

HeaderValue
Content-Typeapplication/json
AuthorizationBearer <VENDING_API_KEY>

Request Body

FieldTypeDescription
qr_token requiredstringThe full QR code string scanned from the staff member's phone
amount requirednumberPositive amount to deduct (e.g. 2.50)
machine_id optionalstringIdentifier for the vending machine (e.g. "VM-01"). Max 50 chars.
description optionalstringItem description (e.g. "Canned Coffee"). Max 120 chars. Shown in user's history.

Example Request

POST /cafeteria/api/vending/deduct/
Authorization: Bearer your-secret-api-key
Content-Type: application/json

{
  "qr_token":    "VEND:EMP-001.a1b2c3d4...",
  "amount":      2.50,
  "machine_id":  "VM-LOBBY-01",
  "description": "Canned Coffee"
}

Success Response (200 OK)

{
  "success": true,
  "balance": 47.50,
  "staff_id": "EMP-001",
  "transaction_id": 42
}

Error Responses

Insufficient funds (200)

{
  "success": false,
  "error":   "insufficient_funds",
  "message": "Insufficient credit. Balance: S$1.20, required: S$2.50",
  "balance": 1.20
}

Invalid QR code (200)

{
  "success": false,
  "error":   "invalid_qr",
  "message": "Invalid or unrecognised QR code"
}

Unauthorized (401)

{
  "success": false,
  "error":   "unauthorized",
  "message": "Invalid API key"
}

Bad request (400)

{
  "success": false,
  "error":   "bad_request",
  "message": "qr_token is required"
}

Error Codes Summary

Error CodeHTTP StatusDescription
unauthorized401Missing or invalid API key
bad_request400Missing required fields or invalid JSON
invalid_qr200QR code is not recognised (tampered, wrong format, or inactive user)
insufficient_funds200User's credit balance is less than the requested amount
server_error500VENDING_API_KEY not configured on the server

Integration Flow

  1. Staff opens their PWA Home page — their unique QR code is displayed beside the credit balance
  2. Staff holds their phone up to the vending machine's QR scanner
  3. Vending machine reads the QR string (format: VEND:<staff_id>.<hmac>)
  4. Vending machine calls POST /cafeteria/api/vending/deduct/ with the QR token and item price
  5. API verifies the QR signature, checks balance, deducts credit atomically
  6. Machine receives response — dispenses item on success, shows error on failure
  7. Transaction appears in staff's History → Vending tab (success or failed with reason)

Testing with cURL

# Deduct S$2.50 from a staff member
curl -X POST {{ base_url }}/cafeteria/api/vending/deduct/ \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "qr_token": "VEND:EMP-001.abc123...",
    "amount": 2.50,
    "machine_id": "VM-LOBBY-01",
    "description": "Canned Coffee"
  }'
